## 0. Scope & principles
Edward reuses PDF templates (certificates, agendas, offer letters). The invisible in-place edit is the soul of the app: NOTHING in this WS touches the bake pipeline (`_apply_page_edits` document.py:1733, `_reinsert_edit` :1882, `save_as` :1670). All new mutations route through the EXISTING command kinds (`move`/`style`/`delete`/`add`, BoxCommand main_window.py:501) so the lockstep undo invariant (main_window.py:398) and screen==saved parity hold by construction. New model APIs are read-only (`page_words`) plus one history-only fuse (`coalesce_last_undo`). Bake/save behavior: byte-identical to today for every feature below.

## 6. Undo / Save / WYSIWYG summary (whole WS)
- Mutations and their commands: nudge/drag → BoxCommand("move") (nudges merge via `coalesce_last_undo`, lockstep preserved); B/I, Paste Style → BoxCommand("style"); canvas paste → BoxCommand("add") (+EditRun on later text change); Cut → BoxCommand("delete"); editor copy/paste participate in the normal commit commands. Structural-boundary wipe (main_window.py:2519) tolerated by all — they are plain `_ModelCommand`s.
- Save: `save_as` untouched; nothing new is staged outside the existing Edit/NewBox maps; text-select and page_words are pure reads of the bake output.
- WYSIWYG: `render_with_edits` untouched; `page_words` shares its pipeline, so the copy tool can never disagree with the screen or the saved file. Existing parity tests (test_editor_model.py:578, test_reflow.py:260) remain the gate.

## 7. DE-SCOPE (deliberate)
Cross-page selection drags (continuous-view ordering ambiguity; single-page covers template copy needs). Char-level (sub-word) selection granularity — word snapping only; rawdict char bboxes exist if it's ever wanted. Hard line breaks / multi-line NewBox frames (NewBox is single-line by model contract, document.py:352; a frame box kind belongs to an objects WS). RTL/bidi caret movement (font engine is LTR-only, font_engine pain points). Pasting images or rich HTML attributes beyond bold/italic (style-sanity rule). Find-match range highlight, smooth Ctrl+wheel zoom, baked-pixmap cache (other workstreams; see §8). Native macOS context-menu key (no Menu key on Mac; right-click only).
