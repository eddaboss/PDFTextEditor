# ws5_forms: AcroForm detect, fill, navigate, flatten

Workstream of the Acrobat-clone buildout. Target user: Edward, who reuses PDF templates
(certificates, agendas, intake forms). Real AcroForm PDFs must become fillable in-app;
documents without forms must look exactly as they do today (zero new chrome).

Stack lock: PySide6 + PyMuPDF 1.27.2.3 (import fitz) + stdlib. No new pip installs.

## 1. Architecture

Form values follow the soul of the app: the working doc is never mutated by a fill.
Staged values live in a model map and are applied to the FRESH copy that
`render_with_edits` / `save_as` / the structural bake already open, via
`widget.field_value = v; widget.update()`. Screen == file by the same construction as text
edits. Field appearances are regenerated by MuPDF itself, so the FontEngine and the
redact-and-reinsert pipeline are untouched (the fonts map's "AcroForm DR/DA has no
FontEngine bridge" pain point is moot: we never draw field text ourselves).

## 2. Model changes (pdftexteditor/document.py)

New frozen dataclass `FormField` (place near `NewBox`, document.py:352):
`page_index:int, name:str, kind:str` ("text"|"checkbox"|"radio"|"combo"|"listbox"|"signature"|"button"),
`rect:tuple` (x0,y0,x1,y1, unrotated page space), `xref:int`, `value:object` (baseline:
str for text/combo, bool for checkbox, on-state str or "Off" for radio group),
`on_state:str|None` (checkbox/radio kid), `options:tuple` (combo/list `choice_values`),
`flags:int`, `multiline:bool`, `readonly:bool`, `text_fontsize:float`, `max_len:int`.
Properties: `identity = (page_index, "form", name, xref)` (per-widget, for canvas items);
`group_key = (page_index, "form", name)` (per-field, for staging; radio kids share it).

State + accessors:
- `self._form_edits: dict[tuple, object] = {}` in `__init__` (document.py:478 region), plus
  `self._form_fields_cache: dict[int, list[FormField]] = {}`.
- `has_form -> bool(self.working.is_form_pdf)`; `form_field_count -> int(self.working.is_form_pdf or 0)`.
- `form_fields(page_index) -> list[FormField]`: enumerate `self.working[i].widgets()`
  (one FormField PER WIDGET, so each radio kid is its own clickable entry), memoized like
  `spans()` (document.py:589). Checkbox baseline `value = (w.field_value != "Off")`. Radio
  group baseline = on-state of the kid whose `field_value` is not "Off", else "Off".
- `effective_form_value(page_index, field) -> staged _form_edits.get(field.group_key, field.value)`.
- `form_tab_order() -> list[FormField]`: all pages in order, fields in `page.widgets()`
  enumeration order, readonly/button/signature/listbox entries excluded.

Mutation, riding the EXISTING command machinery (document.py:984-1058):
- Module sentinel `_FORM_UNSET`. Add field `form_value: object = _FORM_UNSET` to `_BoxState`
  (document.py:425). `_Command` (448) is reused unchanged; carry the `FormField` in its
  `span` slot so `_command_box_ref` (document.py:1404) can return `(page, FormField)` for
  keys with `key[1] == "form"`.
- `_install_state` (document.py:1043 region) routes `key[1] == "form"` to new
  `_install_form_state(key, state)`: pop `_form_edits[key]` when `state.form_value is
  _FORM_UNSET`, else set it.
- `stage_form_value(page_index, field: FormField, value) -> None`: normalize (text -> str,
  truncated to `max_len` when > 0; checkbox -> bool; radio/combo -> str, combo value must be
  in `field.options` else ValueError). before = current staged state (or UNSET); after =
  `_BoxState(form_value=value)`, or `form_value=_FORM_UNSET` when value equals
  `field.value` baseline (reverting to source drops the entry, mirroring no-op Edits at
  document.py:1031). Push via `_push_command(field.group_key, field, "form", before, after,
  label)`. `undo()/redo()` (1373/1386) then work with zero changes beyond `_command_box_ref`.
- `edit_count` (document.py:1639): add `+ len(self._form_edits)`. `dirty` needs nothing
  (`_push_command` already sets `_dirty`).

Apply pipeline, one helper used everywhere:
- `_apply_form_values(self, target: fitz.Document, only_page: int|None = None) -> None`:
  for each `(page, "form", name) -> value` (skip when `only_page` mismatches), walk
  `target[page].widgets()`; match `w.field_name == name`; radio: only the kid with
  `w.on_state() == value` (set `field_value = value`); text/combo/checkbox: set on every
  matching widget (same-name widgets are one PDF field); then `w.update()`. Pure MuPDF,
  no Qt, safe before or after redactions (probed §0).
- `save_as` (document.py:1670): by the time this lands, ws6 has extracted the bake into
  `_baked_copy()` and ws1 made it resolve through the persistent `self.font_engine` (the
  `engine = FontEngine(out)` line is gone). Call `self._apply_form_values(out)` inside
  `_baked_copy()` right after the fresh copy opens, before the per-page edit loop — ONE seam, so
  save_as / export_text / save_optimized_copy / save_flattened all get form values for free.
  AcroForm survives `out.save(garbage=4, deflate=True)` (and ws6's `_save_kwargs()` encryption).
- `render_with_edits` (document.py:2202): extend the no-work short-circuit so a page with
  staged form values does NOT fall back to the plain render; after opening the fresh copy
  call `self._apply_form_values(out, only_page=page_index)` before `_apply_page_edits`.
  When a page has ONLY form edits (no text edits/new boxes), skip `_apply_page_edits`
  entirely: open copy, apply values, `get_pixmap`. Cost per frame = tobytes (3.5ms at 60p)
  + one `widget.update()` per staged field on that page; no FontEngine, no redactions.
- `_bake_pending_edits` (document.py:2498): widen the early return to
  `... and not self._form_edits`; apply `self._apply_form_values(self.working)` and clear
  `_form_edits` BEFORE the QGuiApplication guard (a form-only bake stays headless).
  Structural ops therefore bake fills into `working` exactly like text edits
  (snapshot/bake/mutate discipline at document.py:2532 unchanged).
- `extract_pages` / `split` / `merge` (document.py:2688/2714/2648): wherever `_bake_doc`
  bakes staged text into the throwaway copy, also call `<owner>._apply_form_values(copy)`
  (for merge, the OTHER doc applies its own staged values). Widgets ride `insert_pdf`
  (probed §0).
- `_invalidate_caches` (document.py:2521): add `self._form_fields_cache.clear()`.
- `save_flattened(out_path)`: identical to `save_as` through the bake, then
  `out.bake(annots=False, widgets=True)` before the atomic temp+`os.replace` write
  (reuse the block at document.py:1720-1729). Non-mutating: no `set_path`, no
  `mark_clean`, staged state untouched. `annots=False` keeps future annotation features
  intact; only widgets flatten.

## 3. Canvas changes (pdftexteditor/ui/page_view.py)

- New constant `Z_FORM_HOTSPOT = 8` — the slot assigned by the ws1 M4b registry (annots=5,
  images=7, forms=8; all below Z_HOTSPOT 10, page_view.py:86). Do NOT renumber existing slots.
  Span hotspots overlap-win by z, so the text-editing soul is untouched; fields sit above image
  hotspots so a field over a background image still wins the click.
- New item `FieldHotspot(QGraphicsRectItem)` (pattern: SpanHotspot, page_view.py:126):
  holds the `FormField`, accepts hover. Paint: always a faint accent wash (alpha ~18) so
  fillable fields are discoverable (Acrobat-style), stronger wash on hover, 1.5px accent
  outline when it is `view._focused_field`. Readonly/button/signature/listbox fields get
  NO hotspot (invisible to interaction; their appearance is in the baked pixmap).
- Hotspot build: register a ws1 M4b page-item factory that, if `document.has_form`, returns
  `FieldHotspot`s for `document.form_fields(layer.page_index)` (tracked on `layer.extra_items`,
  freed by `_dematerialize_page` automatically — do not fork `_materialize_page`). Scene rect:
  map the two rect corners through the existing `_scene_point` (page_view.py:2352); the
  probe in §0 shows widget rects share span coordinate space, so rotation is free.
  No form -> the factory returns []; zero new items.
- Press routing inside ws1 M4c's `_press_select` mode handler (not a new inline branch): if the
  topmost item is a `FieldHotspot` (checked AFTER SpanHotspot/_ResizeHandle/SelectionOverlay so
  the box invariant stands), clear box selection, set `_focused_field`, then by kind:
  - checkbox: emit `formCommandRequested(field, {"value": not current_effective})`
  - radio kid: emit with `{"value": field.on_state}` (no toggle-off)
  - combo: `QMenu` of `field.options` at the hotspot; chosen entry emits the command
  - text: mount `FormFieldEditor`
- New signal `formCommandRequested(object, dict)` next to `boxCommandRequested`
  (page_view.py:546).
- `FormFieldEditor(QGraphicsTextItem)`: plain text only, Helvetica `QFont` with
  `setPixelSize(round((field.text_fontsize or 11.0) * zoom))` (pixel sizing per
  font_engine.py:112 precedent), mounted at the field rect's scene top-left with a solid
  white cover rect at Z_COVER under it (field interiors are white; do not use
  `_background_color_for`). Seeds from `effective_form_value`. Keys: Return commits
  (multiline fields: Return inserts newline, Cmd+Return commits), Esc cancels, focus-out
  commits, Cmd+A selects all. Commit emits `formCommandRequested(field, {"value": text})`
  only when changed. Track it in a separate `self._form_editor` slot and flush it
  (commit) at every `_flush_editor` call site (page_view.py:703, 732, 774, 1531, 1562,
  2181) so zoom/reload/page-jump never drop a half-typed value.
- Repaint: `repaint_box(box)` (page_view.py:1462) is called with a `FormField` (it has
  `.page_index` and `.identity`). Verify it re-materializes the page and tolerates an
  identity absent from `layer.boxes`; if the selection-rebind path objects, add a thin
  `refresh_page(page_index)` and prefer it from the command (the window connects
  defensively, main_window.py:1800-1829, so both names are safe).
- Tab navigation (M3): view-level `keyPressEvent` (page_view.py:2207) and
  `FormFieldEditor`: Tab = commit current (if editing) + focus next entry of
  `document.form_tab_order()`, Backtab = previous, wrapping. `focus_field(field)`:
  `scroll_to_page` + ensureVisible on the hotspot rect, set `_focused_field`; text fields
  auto-mount the editor, buttons wait for Space/Return (Space toggles, Return opens
  combo menu). Esc clears field focus.

## 5. Undo / Save / WYSIWYG stories

- Undo: every fill (text commit, checkbox toggle, radio pick, combo pick) is exactly one
  `FormFieldCommand` == one model `_Command` keyed `(page, "form", name)`. Undo restores
  the previous staged value or unstages (baseline shows again). Structural ops bake fills
  into `working` and the Qt stack rebuilds to StructuralCommands, identical to text-edit
  semantics. Flatten export is not undoable (non-mutating export).
- Save: `save_as` applies staged values to the out copy via `widget.update()` and keeps
  the AcroForm fillable; reopen-after-save (the `_reload_after_save` flow,
  main_window.py:2357) naturally discards the stale maps with the old PDFDocument.
  `save_flattened` additionally bakes widgets into page content.
- WYSIWYG: `render_with_edits` applies the same values to the same fresh copy before
  `get_pixmap`, and the probe showed exact ink parity with the saved file. The new-suite
  parity assert (§6) locks it.

## 8. DE-SCOPE

Excluded deliberately: listbox fill (rendered, not interactive; rare in HR templates),
push buttons and JavaScript actions/calculations/format validation (no JS engine in
scope), digital signature SIGNING (signature fields render but are not fillable; crypto
out of scope), free-text entry into editable combos (options menu only), comb-field
cell spacing (filled as plain text), form-field CREATION/design mode, rich-text fields,
honoring an explicit `/Tabs` tab order (widget enumeration order is what Acrobat does
absent one), `NeedAppearances` fallbacks (we always regenerate appearances via
`widget.update()`), and encrypted-PDF handling (the constructor crash at document.py:487
belongs to the document-security workstream).

## 10. Performance notes

Fill commit cost = existing `render_with_edits` page re-bake + one `widget.update()` per
staged field on that page (no FontEngine, no redaction work for form-only pages). The
faint field wash adds one QGraphicsRectItem per widget, only on form docs, only on
materialized pages. `form_fields` is memoized per page and invalidated with the existing
generation bump. The ws1 baked-pixmap cache is live by now: fold the page's sorted
`_form_edits` items into `PDFDocument.render_signature` (the ws1 M1d registry) in M1,
and assert a cache miss after `stage_form_value` in test_forms.
