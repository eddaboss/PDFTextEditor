## GLOBAL INVARIANTS (the soul — violating any of these fails the workstream)

1. **Invisible in-place text edits.** The redact-and-reinsert pipeline
   (`_apply_page_edits`, `_reinsert_edit`) is the heart of the app. No feature
   may degrade font fidelity, introduce base-14 substitutions where embedded
   faces exist (`new_base14 == []` pattern), or leave residue around edited
   boxes.
2. **WYSIWYG: screen == saved file, by construction.** All staged state (text
   edits, new boxes, annots, images, form values) bakes through the ONE shared
   pipeline that serves `render_with_edits`, `save_as`, and the structural
   bake. Never add a second render path. Parity gate: rendered-ink rel diff
   < 0.02 between `render_with_edits` and the saved file's render.
3. **`self.working` is never mutated except by structural ops and bakes.**
   Everything else stages in model maps and applies to fresh copies.
4. **Undo lockstep.** One user action == one Qt command == one model command
   (or one documented macro). New mutation kinds ride the existing
   command/snapshot discipline; structural ops keep their bake-and-wipe
   semantics.
5. **All suites green, every milestone.** Cumulative set; no silent regressions
   of test_redesign / test_text_tools.
6. **Fixtures are synthetic only.** Generated by scripts in `tests/fixtures/`,
   fake neutral names only ("Jordan Carter", "Riley Morgan", "Acme Corp").
   NEVER copy any PDF from outside the repo. Tests never write into
   `tests/fixtures/`; temp output via `tempfile.mkdtemp`.
7. **No modal `exec()` in offscreen tests.** Every dialog gets an injectable
   seam (`_do_*` methods, factories, providers).
8. **Packaging stays intact.** All new modules live under `pdftexteditor/` so
   `collect_submodules` in PDFTextEditor.spec auto-bundles them; no new data
   files, no spec edits expected; ws6 M4 runs `build_app.sh` as a smoke gate.
9. **No AI attribution anywhere** — code, comments, commits, docs. Local-only,
   no telemetry, no cloud.

## CROSS-WORKSTREAM CONTRACTS (established by ws1, consumed by everyone)

- **`PDFDocument.render_signature(page)`** is the single cache-key registry.
  Every workstream that adds staged per-page state folds its sorted state into
  the tuple in the same change (ws3 annots/overrides, ws4 images/xim-deletes,
  ws5 form values) and asserts a pixmap-cache miss in its own suite.
- **Menu registry**: 7-menu skeleton (File/Edit/View/Tools/Document/Window/
  Help) + `self.menu_anchors` separators, implemented by ws1 M4d from the table
  in ws7 §M2 (which remains the registry of record). No workstream adds a
  top-level menu; insert at anchors only.
- **Z-slot registry** (page_view.py constants block): Z_TEXT_SELECT=2 (ws2),
  Z_ANNOT_HOTSPOT=5 (ws3), Z_IMAGE_HOTSPOT=7 (ws4), Z_FORM_HOTSPOT=8 (ws5);
  6 stays Z_PREVIEW_TEXT; text hotspots at 10 always win clicks.
- **Tool-mode dispatch** (`_mode_handlers`, ws1 M4c): all new canvas modes
  (select_text, markup_*, note, ink, shape_*, place_image, crop) register
  handlers; no new inline branches in mousePressEvent/keyPressEvent.
- **Page-item factory** (ws1 M4b): overlay-style per-page items (ws3
  AnnotHotspot, ws5 FieldHotspot) come from registered factories tracked on
  `layer.extra_items`. Exception: ws4's ImageHotspot is a BOX kind and lives in
  the layer `_boxes` identity machinery instead (documented in ws4 §3).
- **Bake order inside `_apply_page_edits`** (per page): ws4 xim-delete
  redaction pass -> text redaction -> reinsertion (one ws1 Shape) -> NewBoxes
  -> ws4 image insertion -> ws3 `_apply_page_annots` LAST. ws5
  `_apply_form_values` runs on the fresh copy outside the per-page tail
  (probe-proven safe before or after redactions).
- **Constructor order** (document.py __init__): password gate (ws6 M4) ->
  insert_pdf -> metadata carry (ws6 M1) -> TOC carry (ws6 M1, absorbed from
  ws7).
- **`edit_count` is the single staged-state census.** ws2 M1 makes
  `undo()/redo()` derive `_dirty` from it; ws3/ws4/ws5 MUST add their staged
  counts to `edit_count` (their specs already do) or the Save button lies.
- **Save funnel**: ws6 extracts `_baked_copy()` (preserving ws1's persistent
  FontEngine); ws5 hooks `_apply_form_values` into that one helper.

   ws4 and ws5 BOTH claimed 8. DECISION: registry in ws1 M4b — text-select 2,

   factory. DECISION: ws3 AnnotHotspot and ws5 FieldHotspot use the factory

## PER-WORKSTREAM COMMIT RITUAL

1. Read the spec end to end; re-find line anchors against CURRENT code (line
   numbers drift as workstreams land — anchors are hints, not gospel).
2. Land milestones in spec order; after each milestone run the FULL cumulative
   suite set.
3. After the last milestone: run every suite once more, then commit with a
   plain descriptive message (no AI attribution), one commit per workstream
   minimum (per-milestone commits welcome).
4. Update `tests/fixtures/manifest.md` idempotently when adding fixtures; new
   suites join the cumulative gate list for everything after them.
